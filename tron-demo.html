 PKG_TRON_INDEXER — ingesta y consulta

      
Package que concentra la lógica de indexación: tipos RECORD/TABLE para carga masiva, ingest_block (JSON_TABLE + FORALL + MERGE), get_wallet_summary (SYS_REFCURSOR) y token_volume.

 Arquitectura — del bloque TRON a la consulta

      

        

          
 TRON / TronWeb

          
TronWeb (Node) se suscribe a nuevos bloques y a eventos Transfer del contrato TRC-20. Arma el JSON del bloque y lo envía a Oracle.

        

        

        

          
 ORDS REST

          
Endpoint REST publicado con Oracle ORDS cuyo handler PL/SQL recibe el CLOB y delega en el package del indexer. Idempotente vía MERGE.

        

        

        

          
 Oracle PL/SQL

          
PKG_TRON_INDEXER parsea con JSON_TABLE y carga con BULK/FORALL. PKG_AML_SCORING recalcula riesgo. DBMS_SCHEDULER corre el batch de catch-up.

        

        

        

          
 Consumo

          
Explorer, dashboards analíticos y alertas AML consultan vistas y funciones del package. Reportes con JasperReports / APEX.

        

      

      

        Por qué PL/SQL para esto: el volumen on-chain es de alta transaccionalidad — millones de filas/día. Procesar el set dentro de la base (set-based SQL + BULK/FORALL) evita el ida y vuelta fila por fila, respeta la integridad referencial y deja la lógica de negocio (scoring AML, agregaciones) donde están los datos. Es el mismo principio de los sistemas críticos en los que trabajé: Oracle como motor, no como simple almacén.
